  • LUPO

  • Farce Book

  • Lair BnB
    Lair BnB is a basic Air BnB clone app with a Lord Of The Rings Theme. This app is the first Makers Bootcamp group engineering project written in Ruby and using Sinatra and PostgreSQL. Users can offer spaces and receive bookinngs in addition to making bookings themselves.
  • Knot Very Useful
    Knot Very Useful is a surprisingly useful educational app with video tutorials demonstrating how to tie the most common Forest School knots. The app is written in vanilla JavaScript, HTML and CSS and was created both as a practical teaching tool and as a code exploration.
  • Picture Pad
    Picture Pad is a joyful exploration of programming that I created during the summer prior to enrolling on Makers Software Development Bootcamp. Written in vanilla JavaScript, HTML and CSS, it was an exploration of flexbox and css to achieve a playful display of photos.
  • Chitter App
    Chitter is a simple Twitter clone app written in Ruby and using Sinatra and PostgreSQL. Although fairly basic in scope, Chitter was the first solo engineering challenge on the Makers Software Development Bootcamp to feature a full user interface.

London Underground Phony Orchestra

The sonification and visualisation of the London Underground Tube networkSonification and visualisation of the London Underground Tube network

Description: A generative music app based on real-time arrivals of trains on the London Underground network using data from the TFL Unified API. Each tube line is assigned an instrument and each station assigned a different note. LUPO is the outcome of my final group engineering project on the Makers Software Development bootcamp. The app was designed and coded entirely from scratch as a group collaboration. It was designed and built in eight days by five students.

Technologies: The app is primarily built in JavaScript using the React.js library. We relied heavily on tone.js for the audio side of things which was new to all in the group. We also learnt chart.js when we discovered an urgent need to visualise TFL's dubious data. We, of course used Git and GitHub throughout as well as Postman.
